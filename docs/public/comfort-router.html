Observable Comfort Router

Routing apps return the shortest path. They don't measure what the walk feels like. This system does — camera-derived street pressure, satellite heat load, and tree shade from public data, on a full Manhattan island walk graph. Five cost encodings on the same graph, same solver: different paths because the cost function changes.

      Graph: manhattan_v1 — full Manhattan island pedestrian walk graph, 4,567 nodes.
      Data surface: Midtown corridor (42nd–59th St, 3rd–9th Ave), where camera stress and thermal data have been assembled.
      Routing weights computed at graph-build time from July 2024 Landsat LST composite. The time-of-day selector updates the thermal viewer layer; routing weights are static. This gap (the route uses build-time data, not request-time data) is the primary finding of the companion paper.
      

      Heat routing uses Landsat surface temperature on the street graph (greenR framework, Mahajan Ecological Indicators 2024).
      Surface temperature is not pedestrian-height air temperature; on parallel avenues the difference a walker feels is governed by upwind conditions the satellite does not see.
      Thermal layers honesty: MRT is modeled (static LST composite, not per-bin surface temperature); UTCI is provisional (proxy air temp, proxy wind, proxy humidity); Central Park canopy height is a 12 m DBH proxy, not survey.
      Deferred tools (not yet implemented): fetch_environmental (live weather at routing time), fetch_instruments (sensor coverage per edge). Minimal closure: one API call per request.
