wavves

durable multi-agent routing

wavves turns moderator intent into alignment packets, model recommendations, check records and handoff files so distributed agent work has a file-backed route instead of a long chat as its only memory.

`INDEX.md` is the fast pickup map. `AGENTS.md` is the standing contract. Lane homes carry the work, evidence and model choices. Rotation files assign successor identity when the moderator thread gets heavy.

Model Routing

The moderator records model recommendations before dispatch. The goal is not to claim savings before measurement. The goal is to reserve high-reasoning models for judgment-heavy work and make the decision auditable.

RoleRecommended tierReason
Lane orchestratorhigh-reasoningCross-file plan, acceptance criteria and gate design.
Discovery runnersfastInventory, search, link checks and mechanical scans.
Build runnersbalancedBounded edits with local validation.
Adversarial gatehigh-reasoningRisk, defect finding and caveat review.
Acceptance gatehigh-reasoningFinal verification with captured evidence.

When Cursor exposes a model field for a subagent, the moderator can set it during dispatch. When model selection is unavailable, the recommendation stays in the prompt and the runner reports the enforcement gap.

Walkthrough

  1. Set up `wavves/` once per repo.
  2. Write a `waveset.md` with scope, locks, gates and model recommendations.
  3. Dispatch the lane orchestrator with `dispatch.md`.
  4. Run wave members against disjoint files.
  5. Capture gate evidence under `gate-captures/`.
  6. Rotate through `rotations/` when the moderator thread gets heavy.
